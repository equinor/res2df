res2df.parameters.find_parameter_files(deckpath: res2df.resdatafiles.ResdataFiles | str | pathlib.Path, filebase: str = 'parameters') → list[pathlib.Path][source]

Locate a default prioritized list of files to try to read as key-value

File extensions .yml, .json and .txt are recognized and will be found in current dir, one directory up, and two directories up.

Parameters:

  • deckpath – Either a ResdataFiles object of a simulator output set (only the corresponding path will be used), or path to a file or directory, that will be used as a starting point for locating parameter files

  • filebase – the base of filenames to look for.

Returns:

Absolute paths to filenames. Empty list if nothing found

res2df.parameters.load_parameterstxt(filename: str | pathlib.Path) → dict[str, Any][source]

Read parameters.txt into a dictionary

Lines starting with a hash will be ignored.

Parameters:

filename – file containing one key-value pair pr. line, separated by whitespace

res2df.parameters.load_all(filenames: list[str] | list[pathlib.Path], warnduplicates: bool = True) → dict[str, Any][source]

Reads a list of parameter filenames

Dictionaries for all files will be merged into one.

Keys must be unique over all filenames, if not only the first occurence will be used (based on the order of filenames).

Parameters:

  • filenames – Order matters.

  • warnduplicates – If True (default), overlapping keys will be warned.
