ecl2df.equil

Extract EQUIL from an Eclipse deck as Pandas DataFrame

ecl2df.equil.df(deck, keywords=None, ntequl=None)[source]

Extract EQUIL related keyword data, EQUIL, RSVD, RVVD PBVD and PDVD.

How each data value in the EQUIL records are to be interpreted depends on the phase configuration in the deck, which means that we need more than the EQUIL section alone to determine the dataframe.

If ntequl is not supplied and EQLDIMS is not in the deck, the equil data is not well defined in terms of OPM. This means that we have to infer the correct number of EQUIL lines from what gives us successful parsing from OPM. In those cases, the deck must be supplied as a string, if not, extra EQUIL lines are possibly already removed by the OPM parser in eclfiles.str2deck().

Parameters:

  • deck (Union[str, EclFiles, Deck]) – Eclipse deck or string with deck. If not string, EQLDIMS must be present in the deck.

  • keywords (Optional[List[str]]) – Requested keywords for which to extract data.

  • ntequl (Optional[int]) – If not None, should state the NTEQUL in EQLDIMS. If None and EQLDIMS is not present, it will be inferred.

Return type:

DataFrame

Returns:

pd.DataFrame, at least with columns KEYWORD and EQLNUM

ecl2df.equil.df2ecl(equil_df, keywords=None, comments=None, withphases=False, filename=None)[source]

Generate Eclipse include strings from dataframes with solution (EQUIL, RSVD++) data.

Parameters:

  • equil_df (DataFrame) – Dataframe with data on ecl2df format.

  • keywords (Optional[List[str]]) – List of keywords to include. Must be supported and present in the incoming dataframe.

  • comments (Optional[Dict[str, str]]) – Dictionary indexed by keyword with comments to be included pr. keyword. If a key named “master” is present it will be used as a master comment for the outputted file.

  • withphases (bool) – If True, the phase configuration keywords will be outputted. This is mostly for testing, and should not be necessary in production.

  • filename (Optional[str]) – If supplied, the generated text will also be dumped to file.

Return type:

str
