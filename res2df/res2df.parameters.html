res2df.parameters

Support module for extra files with key-value information related to simulator runs

res2df.parameters.find_parameter_files(deckpath, filebase='parameters')[source]

Locate a default prioritized list of files to try to read as key-value

File extensions .yml, .json and .txt are recognized and will be found in current dir, one directory up, and two directories up.

Parameters:

  • deckpath (Union[ResdataFiles, str, Path]) – Either a ResdataFiles object of a simulator output set (only the corresponding path will be used), or path to a file or directory, that will be used as a starting point for locating parameter files

  • filebase (str) – the base of filenames to look for.

Return type:

List[Path]

Returns:

Absolute paths to filenames. Empty list if nothing found

res2df.parameters.load(filename)[source]

Read a parameter file as txt, yaml or json

Return type:

Dict[str, Any]

Returns:

dict() with parameter names as keys. Empty dictionary if

no parameters in the file.

Raises ValueError or IOError if no files are readable

res2df.parameters.load_all(filenames, warnduplicates=True)[source]

Reads a list of parameter filenames

Dictionaries for all files will be merged into one.

Keys must be unique over all filenames, if not only the first occurence will be used (based on the order of filenames).

Parameters:

  • filenames (Union[List[str], List[Path]]) – Order matters.

  • warnduplicates (bool) – If True (default), overlapping keys will be warned.

Return type:

Dict[str, Any]

res2df.parameters.load_parameterstxt(filename)[source]

Read parameters.txt into a dictionary
