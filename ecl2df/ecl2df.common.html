ecl2df.common.ecl_keyworddata_to_df(deck, keyword, renamer=None, recordcountername=None, emptyrecordcountername=None)[source]

Extract data associated to an Eclipse keyword into a tabular form.

Two modes of enumeration of tables in the keyworddata is supported, you will have to find out which one fits your particular keyword. Activate by setting recordcountername or emptyrecordcountername to some string, which will be the name of your enumeration, e.g. PVTNUM, EQLNUM or SATNUM.

Parameters

  • deck – Parsed deck

  • keyword (str) – Name of the keyword for which to extract data.

  • renamer (Optional[Dict[str, Union[str, List[str]]]]) – Mapping of names present in OPM json files for the keyword to desired column names in returned dataframe

  • recordcountername (Optional[str]) – If present, an extra column is added with this name with consecutive rows enumerated from 1. Use this to assign EQLNUM or similar when this should be consecutive pr. row (not the case for all keywords).

  • emptyrecordcountername (Optional[str]) – If supplied, an index is added to every parsed row based on how many empty records is encountered. For PVTO f.ex, this gives the PVTNUM indexing.

Return type

DataFrame

ecl2df.common.get_wells_matching_template(template, wells)[source]

Returns the wells in the list that is matching the template containing wilcard characters. The wildcard charachters supported are * to match zero or more charachters and ? to match a single non-empty character. Any wildcards in the beginning of the template must be preceded with a .

Well name templates starting with ? might be allowed in Eclipse in some contexts, but not in all and will not be permitted here to avoid confusion.

Parameters

  • template (str) – well name template with wildcard characters

  • wells (list) – list of wells to match against the template

Returns

List of matched wells

ecl2df.common.parse_lyrfile(filename)[source]

Return a list of dicts representation of the lyr file.

The lyr file contains data of the following format, where the color code is optional:

'ZoneA' 1-4     #FFE5F7
'ZoneB' 5       red

Description of the lyr format: https://resinsight.org/3d-main-window/formations/#formation-names-description-files-_lyr_

The output has the following format:

[
    {
        "name": "ZoneA",
        "from_layer": 1,
        "to_layer": 4,
        "color": "#FFE5F7"
    },
    {
        "name": "ZoneB",
        "span": 5,
        "color": "red"
    }
]
Parameters

filename (str) – Absolute path to a lyr file

Return type

Optional[List[Dict[str, Any]]]

Returns

A list of dictionaries representing the information in the lyr file.

ecl2df.common.runlength_eclcompress(string, sep='  ')[source]

Compress a string of space-separated elements so that

2 2 2 2 2 3 3 4

becomes

5*2 2*3 4

which is the format supported by Eclipse. The input string must be splittable with split(). Any newlines will be replaced by a space prior to split().

See https:///en.wikipedia.org/wiki/Run-length_encoding

Parameters

  • string (str) – String of space-separated elements to compress

  • sep (str) – Separator string used in output. Separator string in input is not conserved, but it must be compatible with str.split()

Return type

str

Returns

string, shorter or equal-length to the input.

ecl2df.common.stack_on_colnames(dframe, sep='@', stackcolname='DATE', inplace=True)[source]

For a dataframe where some columns are multilevel, but where the second level is encoded in the column name, this function will stack the dataframe by putting the second level of the column multiindex into its own column, best understood by this example:

A dframe like this

PORV

OWC@2000-01-01

OWC@2020-01-01

100

1000

990

will be stacked to

PORV

OWC

DATE

100

1000

2000-01-01

100

990

2020-01-01

(for the defaults values for sep and stackcolname)

Column order is not guaranteed

Parameters

  • dframe (DataFrame) – A dataframe to stack

  • sep (str) – The separator that is used in dframe.columns to define the multilevel column names.

  • stackcolname (str) – Used as column name for the second level of the column multiindex

Return type

DataFrame
